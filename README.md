## вњЁ Features

### рџЏ  Core Platform
- **Authentication** вЂ” JWT in httpOnly cookies with Redis-based logout blocklisting
- **Problem Library** вЂ” Filterable, searchable, sortable problem list with difficulty, tags, and points
- **Monaco Editor** вЂ” Full-featured code editor with C++, Java, JavaScript support, syntax highlighting, and per-language code buffers

- **Progress Tracking** вЂ” Solved problems tracked per user with stats dashboard

### вљ”пёЏ 1v1 Code Duel (Real-time)
- **Room-based matchmaking** вЂ” Host creates a room with a shareable 6-character code
- **Configurable battles** вЂ” Host sets time limit (presets + custom) and number of questions (1вЂ“5)
- **Points system** вЂ” Each problem has admin-set points, most points wins
- **Live opponent status** вЂ” See opponent's test progress in real time (no code sharing)
- **Early end detection** вЂ” Battle ends instantly when a winner is mathematically certain
- **ELO rating system** вЂ” K=32 ELO updates after every duel, leaderboard on lobby page
- **Taunt system** вЂ” Send fun emojis to your opponent mid-battle
- **Post-battle recap** вЂ” View both solutions side by side after the match

### рџ¤– AI Features
- **AI Code Review Agent** вЂ” After every submission, a ReAct-pattern agent runs your code via Judge0, reasons about failures, generates a fix, verifies it, and returns structured feedback (complexity, strengths, improvements, cleaner approach)
- **AI Hint System** вЂ” 3 progressive hints per problem generated by OpenRouter GPT-4o-mini, cached in Redis for 24hrs so each problem only ever calls the API once
- **AI Tutor** вЂ” Chat-based DSA tutor scoped to the current problem, powered by OpenRouter

### рџ‘¤ Profile & Stats
- **Activity heatmap** вЂ” GitHub-style submission heatmap for the past 365 days
- **Difficulty donut chart** вЂ” Visual breakdown of solved problems by difficulty
- **Streak tracking** вЂ” Current and best solving streaks
- **ELO history** вЂ” Duel rating and duels played count
- **Recent submissions** вЂ” Last submissions with status badges

### рџ› пёЏ Admin Panel
- **Problem creation** вЂ” Full form with title, description, constraints, difficulty, points, tags
- **Test case management** вЂ” Add/remove visible and hidden test cases dynamically
- **Code templates** вЂ” Set starter code and reference solutions for all 3 languages
- **Reference validation** вЂ” Reference solutions are validated against visible test cases via Judge0 before saving (optional вЂ” warns but saves if Judge0 is unavailable)

## рџљЂ Quick Start

### Prerequisites
- **Node.js** v18 or higher
- **Redis** running locally (`redis-server`) or a cloud Redis URL
- API keys for: MongoDB Atlas, Cloudinary, OpenRouter, Judge0 CE

### 1. Backend

```bash
cd backend
npm install
cp .env.example .env
# Fill in all keys in .env
npm run dev
```

Backend runs on **http://localhost:3000**

### 2. Frontend

Open a second terminal:

```bash
cd frontend
npm install
npm run dev
```

Frontend runs on **http://localhost:5173**

The Vite dev server proxies all API routes to `localhost:3000` automatically.

## рџЏ—пёЏ Key Design Decisions

- **Socket.io auth** uses JWT passed via `socket.handshake.auth.token` вЂ” fetched fresh from `/user/get-token` since httpOnly cookies aren't readable by JS
- **Redis** stores duel room state with TTL вЂ” MongoDB only gets written when a duel finishes (permanent record)
- **ELO** uses K=32, standard chess formula вЂ” draw = no ELO change
- **Hint caching** вЂ” all 3 hints generated in one API call, cached for 24hrs per problem in Redis
- **Agent loop** вЂ” max 4 iterations to prevent infinite loops, `<RUN_CODE>` tags signal tool use
- **Theme** вЂ” synced via `MutationObserver` on `document.documentElement` class changes so all pages react instantly to toggle
